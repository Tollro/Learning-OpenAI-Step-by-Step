# Prompt提示词工程化

1. 角色：角色设定、角色背景或能力
2. 场景需求：需求/目标描述
3. 任务：满足需求/目标的可执行清单
4. 示例：成功案例 、失败反例、格式模板
5. 约束：显性限制/红线 + 偏好 + 风险规避/敏感项

Zero-shot\few-shot

CoT（chain of thought）思维链：推理具有优势

​	通过引导提示，生成中间推理步骤，基于中间步骤得出答案。

ToT（Tree of thought）思维树：数学题具有优势

​	核心在于将问题的解决过程视为一颗思维树。

·  搜索策略:广度优先搜索（BFS）、深度优先（DFS）、蒙特卡洛树

​                                  ↑                                  ↑		

​                并行扩展，计算成本高       深入单一路径

·  节点评估、路径选择：自我评估（概率输出）、外部验证（规则、工具调用、人工反馈）  

·  回溯、全局优化：路径陷入死胡同是返回父节点，尝试其他分支；对模糊节点引入蒙特卡洛采样，多次评估量化不确定性。
